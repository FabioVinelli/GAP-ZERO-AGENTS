## Terminal contract
Every task ends in exactly one of: executed action (landing-checked) | validated handoff | explicit block.
A bare report is a failure mode. NEVER claim an action landed without landing-check evidence.
NEVER simulate an action and present it as executed — label all dry runs "SIMULATED — NOT LANDED".

## Outcome (L0)
- Metric: % toolset verification checks passing with landed evidence
- Baseline → target: 0% → 100%
- Owner (accountable for errors): Ko

## Pipeline (insight → action)
Cycle Trigger → Env Fingerprint → File Round-Trip → Shell Exec Test → Structured Echo → Ko Review ◇ → Run Summary Landed

## Policy rules (pre-execution, enforced by src/gapzero/policy.py)
- local reversible file/subprocess inside agent dir: ALLOW
- write operation outside agent directory: BLOCK
- network call or credential access attempt: ESCALATE
- escalation event itself — occurrence is test failure: BLOCK
- Irreversible actions ALWAYS escalate to the human touchpoint. No exceptions, no self-issued approval tokens.

## Human touchpoint (max ONE validation step)
- Design: Single post-cycle review by Ko: receives run summary with pass/fail per check, policy log, JSONL trace path. Cycle auto-halts on any landing-check failure or policy escalation before Ko review.
- Owner: Ko · Detection: Any landing-check assertion failure or policy escalation sets cycle_status=FAILED in run summary; summary written before human review; Ko reviews artifact not live state · Rollback: Delete agent test-output directory entirely; no writes exist outside it; single rm -rf; reversible with zero side effects
